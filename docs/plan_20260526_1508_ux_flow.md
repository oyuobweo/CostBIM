# CostBIM 프리미엄 UX 동선 개편 및 탭 한줄 정렬 보정 계획서 (v3.0)

본 계획서는 마스터 룰(Lead Engineer Master Rules v2.1)에 의거하여, 사용자가 제시한 추가 피드백(탭 한 줄 정렬 보장, 불필요한 전체 선택/해제 버튼 삭제, 매개변수 설정 직후 즉각 스캔이 가능한 Parameter Scan 버튼 동선 배치)을 반영하여 완벽한 실무 대시보드 인터페이스를 완수하기 위한 정밀 튜닝 설계도입니다.

---

## 1) Problem Summary (핵심 문제 요약)
1. **탭 헤더 2줄 래핑 현상**: 탭 가로 패딩(`22px`) 확장으로 인해 한정된 우측 패널 가로 너비(기존 260px) 내에서 탭들이 한 줄에 담기지 못하고 두 줄로 꺾이는 이질감 해결 필요.
2. **불필요한 선택 툴바 버튼 노출**: 전체 선택, 전체 해제 버튼이 오히려 화면에 시각적 노이즈를 주므로 이를 완전히 삭제하여 미니멀리즘 강화.
3. **2차 스캔 실행 버튼의 부재**: 데이터 스캔 완료 후 `EmptyStatePanel`이 Collapsed 되면, 화면 중앙의 스캔 버튼이 사라져 사용자가 매개변수를 새로 체크한 후 다시 스캔할 수 있는 동선(버튼)이 부재한 심각한 UX 결함 해결 필요.

---

## 2) Design Summary (디자인 개요 및 주요 모듈)
* **탭 한 줄 정속 100% 정렬**:
  - 우측 매개변수 설정 세로 패널의 가로 폭(Column 2)을 `260px`에서 **`285px`**로 확장하여 가로 캔버스 용량 증대.
  - `PremiumTabItemStyle`의 `Padding`을 `22,8,22,8`에서 **`14,8,14,8`**로 미세 조율하고 탭 간 마진을 `Margin="0,0,4,-1"`로 튜닝하여 탭 헤더들이 소수점 오차 없이 완벽하고도 편안하게 한 줄로 가로 정렬되도록 보증.
* **불필요한 버튼 소거**: 전체 선택/전체 해제 StackPanel 영역을 완전히 걷어내어 여유 있는 툴바 공간 확보.
* **상시 스캔 버튼 UX 동선 재구축 (Parameter Scan)**:
  - 우측 `ParameterConfigPanel` (Column 2)의 Grid RowDefinitions에 맨 하단 Row를 신설(`Height="Auto"`).
  - 매개변수를 조작(체크/체크해제)한 직후 즉각 손가락 동선이 닿는 탭 리스트 맨 아래 영역에 **상시 노출형 `Parameter Scan`** 프리미엄 버튼(`BtnRightScan`) 배치.
  - 우측 하단 공용 버튼 바에 있던 기존 실행 버튼(`BtnExtract`, "스캔") 또한 이모지 Path를 걷어내고 텍스트를 **`Parameter Scan`**으로 완벽하게 동기화하여 일관성 있는 듀얼 실행 동선 확보.

## 3) Proposed Changes (제안된 변경 내용)

### [Views/MainWindow.xaml](file:///d:/CostBim/Views/MainWindow.xaml)

* **우측 패널 가로 너비 확장**:
  - 데이터 탭 내부 `Grid.ColumnDefinitions` (890 라인 부근)에서 Column 2의 가로 너비 `Width="260"`을 **`Width="285"`**로 넓혀 동선 버튼과 탭 한 줄 정렬 공간 확보.
* **TabItem 스타일 가로 패딩 보정**:
  - `PremiumTabItemStyle` (416 라인 부근)의 `Padding`을 `22,8,22,8`에서 **`14,8,14,8`**로 미세 보정하여 285px 패널 내 한 줄 정렬 충족.
  - `Border Margin`을 `0,0,4,-1`로 튜닝하여 정속 밀착.
* **전체 선택 / 전체 해제 버튼 삭제**:
  - `ParameterConfigPanel` (991 라인 부근) 내부의 `Selection Toolbar` Grid 내 StackPanel(전체 선택, 전체 해제 버튼)을 완전히 삭제하고, 상단 정렬 체크박스만 가시성 있게 유지.
* **우측 패널 하단 상시 실행 버튼 (`Parameter Scan`) 신설**:
  - `ParameterConfigPanel` Grid RowDefinitions를 4단(`Height="Auto"`, `Height="Auto"`, `Height="*"`, `Height="Auto"`)으로 신설.
  - `Grid.Row="3"` 영역에 널찍하고 세련된 상시 노출형 **`Parameter Scan`** 버튼(`BtnRightScan`) 추가.
* **하단 버튼 바 실행 버튼 명칭 동기화**:
  - 우측 하단 공용 버튼 바의 `BtnExtract` 내 복잡한 Path 이모지를 제거하고, 단순 텍스트 `Parameter Scan`으로 변경 및 텍스트 짤림 방지를 위해 버튼 Width를 `140`으로 유지.

## 4) Verification Plan (검증 계획)
1. **컴파일 빌드 검증**: `dotnet build CostBIM.csproj` 전체 컴파일이 컴파일 에러 없이 빌드 성공하는지 확인.
2. **실물 작동 검증**:
   - 데이터 스캔 완료 후에도 우측 매개변수 설정 탭 바로 아래에 `Parameter Scan` 버튼이 상시 정갈하게 노출되어 클릭 시 즉각 재스캔이 구동되는지 확인.
   - Built-in, Project, Shared 세 개의 탭 헤더가 꺾임이나 두 줄 래핑 현상 전혀 없이 **깔끔하게 한 줄로 좌측 정렬**되어 표출되는지 육안 확인.
   - 전체 선택, 전체 해제 버튼이 삭제되어 시각적 불필요 노이즈가 제거되었는지 확인.

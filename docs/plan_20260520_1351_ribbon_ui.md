# 계획서: CostBIM Revit 리본 전용 버튼 구현 및 UI 노출 정상화

---
> **Role**: 설계 검증, 코드 품질, 운영 안정성을 책임지는 **Lead Engineer Agent**로서 Revit 리본 인터페이스 통합 및 버튼 로드 정상화를 설계합니다.
---

## 1) Problem Summary (문제 요약)
1. 현재 애드인이 `IExternalCommand`로만 등록되어 있어 Revit 로드에는 성공하였으나, 상단 리본 메뉴에 바로 보이지 않고 `Add-ins` 탭 내의 `External Tools` 메뉴 안에만 나타나는 상황입니다.
2. 이를 해결하기 위해 `IExternalApplication`을 구현하여 Revit 실행 시 `Add-ins` 리본 탭에 고유한 `CostBIM` 패널과 프리미엄 3D 분석 아이콘이 탑재된 대형 푸시 버튼(Ribbon Button)을 정식 등록합니다.

## 2) Design Summary (설계 요약)
* **목적**: 
  - Revit 상단 리본바의 `Add-ins` 탭에 `CostBIM` 패널 추가.
  - 프리미엄 브랜드 아이콘(`costbim_icon.png`)이 입혀진 푸시 버튼 생성 및 클릭 시 `CostBIMCommand` 실행되도록 라우팅.
* **주요 구성 요소**:
  - `CostBIMApp` 클래스 (`IExternalApplication` 상속) 추가.
  - `CostBIM.addin` 매니페스트의 어플리케이션 타입 변환 (`<AddIn Type="Application">`).
* **예외 처리**:
  - 이미지 파일 로드 실패 시 버튼 생성 무중단 작동 보장 (Default Revit 아이콘 대체 동작).

## 3) Implementation Plan (구현 계획)
* **단계 1**: `src/CostBIMApp.cs` 파일을 생성하여 `IExternalApplication` 구현 및 `Add-ins` 탭 하위에 푸시 버튼 구성 코드 작성.
* **단계 2**: [CostBIM.addin](file:///D:/CostBim/CostBIM.addin) 매니페스트 설정을 `<AddIn Type="Application">` 및 `FullClassName`을 `CostBIM.CostBIMApp`으로 변경.
* **단계 3**: 프로젝트 빌드 및 `install_addin.ps1`을 가동하여 전 버전 Revit 애드인 폴더에 핫 리로드 배포.

## 4) Testing & Verification Plan (검증 계획)
* **검증 명령어**:
  - `dotnet build d:\CostBim\CostBIM.csproj`
  - `powershell -ExecutionPolicy Bypass -File d:\CostBim\install_addin.ps1`
* **UI 검증**:
  - 빌드 후 설치 시, Revit의 `Add-ins` 탭 하위에 `CostBIM` 패널과 `Extract Parameters` 버튼이 입체적인 프리미엄 3D 아이콘과 함께 표출되는지 검증.

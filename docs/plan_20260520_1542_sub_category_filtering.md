# 설계 계획서: Revit 'CostBIM' 세부 카테고리 가상 매개변수화 및 실시간 필터링 고도화
> **작성일시**: 2026-05-20 15:42
> **역할**: Lead Engineer Agent

---

## 1. Problem Summary (문제 요약)
1. **문제 발생**:
   - 기존 구현에서는 도면 스캔 결과에 "카테고리" 매개변수가 포함되어 있지 않을 경우, 좌측 파라미터 선택 영역에 "카테고리" 항목 자체가 나타나지 않는 치명적인 구조적 한계가 존재함.
   - 이로 인해 사용자는 "아예 안보이는데"라는 피드백을 전달하게 됨.
   - 또한, `RevitElementExtractor.cs` 내에서 "센터라인", "중심선", "그리드", "선" 등의 객체들을 수집 단계에서 원천적으로 강제 제외(`continue`)해 버림으로써, 사용자가 도면에서 실제로 제어하고 싶어하는 수량 외 객체(예: 스코프박스, 센터라인 등)가 표 및 세부 필터에 전혀 등장하지 못하는 문제가 있음.

2. **해결책**:
   - 도면 스캔 결과에 무관하게, 좌측 "Properties" 탭 최상단에 **"카테고리"라는 가상의 매개변수 항목(ParameterItem)을 항상 강제로 생성하여 제공**함.
   - `RevitElementExtractor.cs` 내의 과도한 하드코딩 제외 필터(센터라인, 중심선 등)를 완전히 제거/완화하여 도면 내 모든 가시 객체들이 누락 없이 정상적으로 수집되도록 함.
   - 수집된 객체들을 기반으로 "카테고리" 가상 매개변수 하위에 실제 카테고리(예: 벽, 바닥, 스코프 박스, 센터라인 등)를 **동적 세부 카테고리 필터(SubItems)**로 구축함.
   - 개별 세부 카테고리 체크박스 선택/해제 상태에 따라 우측 표(DataGrid)의 행(Row)들이 0ms 지연으로 실시간 필터 아웃되도록 연동함.

## 2. Design Summary (설계 요약)
1. **목적**:
   - 수량과 무관한 모든 도면 객체들까지 유연하게 수집하되, 사용자가 좌측 "카테고리"의 하위 항목 선택기(Expander)를 통해 "센터라인", "스코프박스" 등을 간편하게 해제하여 표에서 즉각 제외할 수 있도록 함.
2. **입출력 구성**:
   - **입력**: 도면 3D 뷰에서 추출된 전체 객체 데이터 (`List<ExtractedElement>`).
   - **처리**: 
     - "카테고리" 파라미터 하위 `SubItems`에 수집된 고유 카테고리 목록 동적 생성 (기존 체크 상태 사전 보존 딕셔너리 연동).
     - 체크 해제된 세부 카테고리명을 `HashSet<string>`에 수집하여 데이터 필터링(`ApplyFilter`) 시 O(1) 초고속 아웃.
   - **출력**: 실시간 정밀 필터링 처리되어 DataGrid에 렌더링되는 정제된 객체 리스트.
3. **예외 처리 및 안전성**:
   - 도면 재스캔(객체 수집 재실행) 시에도 사용자가 기존에 체크 해제해 둔 세부 카테고리의 상태가 초기화되지 않고 100% 보존되도록 룩업 딕셔너리 연동 보장.

## 3. Implementation Plan (구현 계획)

### [Component 1] UI & ViewModel 고도화 (`MainWindow.xaml.cs`)
- `LoadScannedParameters` 시작 시 `BuiltInParams` 맨 위에 항상 `"카테고리"`라는 이름의 `ParameterItem`을 가상으로 주입 등록.
  - 그룹명: `"기본 정보"`로 강제 설정하여 최상단에 은은하게 위치하도록 함.
- `BuildSubCategoryFilters`가 `UpdateElementsList` 완료 시 가상으로 주입된 `"카테고리"` 파라미터를 정확히 찾아내어 `SubItems`를 빌드하도록 연동.
- 중복 주입 방지를 위해 스캔된 스키마에 동일 명칭의 파라미터가 있을 경우 스킵 처리.

### [Component 2] Revit 객체 추출 필터 완화 (`RevitElementExtractor.cs`)
- `ExtractVisibleElements` 메서드 내부에서 과도하게 걸러내던 하드코딩 제외 로직을 수정함:
  - "센터라인", "중심선", "Centerline", "선" 등의 하드코딩 제외 조건 완전 제거.
  - 사용자가 도면에서 켜고 끌 수 있는 모든 실물 및 참조 모델 요소들이 정상 수집되도록 보장함.

### [Component 3] 실시간 반응형 필터 및 UI 통합 (`MainWindow.xaml`)
- ListBox DataTemplate 내부에 탑재된 Expander와 ItemsControl 기반의 세부 카테고리 체크박스 레이아웃의 바인딩 동작성 최종 확인 및 미세 조정.
- `ApplyFilter` 고도화를 통해 `catParam`이 체크되었든 아니든 하위 `SubItems` 중 해제된 항목들이 표(DataGrid)에서 실시간으로 완벽히 제거되도록 로직 안착.
